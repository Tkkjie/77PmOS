## 觸發詞

下班、end-of-day、收工、日結

## 啟動

1. 讀取 `02-工作管理/CLAUDE.md` 載入規則。
2. 讀取今天的 `02-工作管理/每日待辦/YYYY-MM-DD.md`。
3. 讀取 `02-工作管理/收件匣.md`（避免覆蓋已有項目）。
4. 用 `date +%H:%M` 取得當前時間。

---

## Phase 1：自動掃注記

5. 掃描每日待辦所有區塊（排程 / If Time Allows / Waiting / In Progress），找出：
   - 有 `[x]` 勾選的項目（排程區的任務和會議都算）→ 動作：**完成**
   - 有 `→` 注記的項目 → 按下方模式辨識動作
   - **注意**：Must Do 區只是摘要，不含個別任務。所有打勾和注記都在排程區。

6. **注記辨識規則**（`→` 後的文字，不區分大小寫）：

   | 模式 | 動作 | 任務檔變更 |
   |------|------|-----------|
   | 完成、做完了、done、OK了 | **完成** | status→done, 搬封存 |
   | 封存、算了、取消、不做了、不追了 | **取消** | status→cancelled, 搬封存 |
   | 改[時間詞]前、deadline [時間詞] | **改期** | 更新 deadline + schedule |
   | 沒處理到、沒做、來不及、延期 | **順延** | schedule→明天 |
   | 衍生、下一步是、接下來要 | **衍生** | 原任務照常處理，衍生內容加入收件匣 |
   | 已[動作]，但[剩餘] | **部分完成** | 更新下一步 + schedule 依剩餘判斷 |
   | 私下處理、DM處理 | **順延+備註** | schedule→明天，加備註 |
   | 封存不追蹤 | **取消** | status→cancelled, 搬封存 |

   **時間詞解析**：
   - 明天 → +1 天
   - 後天 → +2 天
   - 週X / 禮拜X → 本週或下週的該天（已過則下週）
   - 這週X前 → 本週該天
   - MM-DD / MM/DD → 指定日期

   **一個注記可能觸發多個動作**：例如「完成，衍生下一步是 XXX」→ 完成 + 衍生。

7. 找到對應的任務檔：
   - 從每日待辦的 tasks.md 或項目文字中找 task ID（T-YYYYMMDD-NNN）
   - 找不到 ID 的，用 Grep 搜任務庫比對任務標題

8. 列出確認表：

   ```
   ## Phase 1：偵測到 N 項有注記/勾選

   | # | 任務 | 注記 | 動作 | 任務檔 |
   |---|------|------|------|--------|
   | 1 | XXX | → 封存 | cancelled → 封存 | T-20260310-053 |
   | 2 | YYY | [x] + → 衍生：ZZZ | done → 封存 + 收件匣 | T-20260322-001 |
   | 3 | AAA | → 改週四前 | deadline→03-26 | T-20260322-012 |

   → 全部 OK？（Y / 逐項調整）
   ```

9. 等用戶確認。用戶可以：
   - 全部 OK → 進入 Phase 2
   - 指定某項調整 → 修正後重新確認
   - 補充遺漏的注記 → 加入清單

---

## Phase 2：互動掃殘餘

10. 找出沒有注記也沒有 `[x]` 的項目，按區塊分組。

11. **排程區**（最重要）：列出未勾選也沒注記的任務項（不含會議），一次問完。
    ```
    排程還有 N 項任務沒勾也沒注記：
    1. 14:00-14:30 綠能訪談 — 訪綱撰寫（3h）
    2. 17:00-17:30 IG 帳號改名（30m）⚡ deadline 明天
    各是什麼狀況？
    ```

12. **If Time Allows 區**：
    - 項目少（<=3）→ 逐項列出問狀況
    - 項目多（>3）→ 問「有做到哪些嗎？還是全部順延？」
    - 用戶說全部順延 → 不逐項問，全部 schedule 不變（本來就是 if time allows）

13. **Waiting 區**：
    ```
    Waiting 有 N 項，有沒有今天解除的？
    - 波士頓科學演講（等 Ginny, 8 天）
    - 瘦瘦筆 V1（等客戶確認, 1 天）
    - ...
    ```
    用戶指出解除的 → 加入動作清單。其餘不動。

14. **In Progress 區**：只確認有沒有變動，沒有就跳過。

15. 用戶回覆後，將所有回覆解析成動作，合併到 Phase 1 的動作清單。

---

## Phase 3：批次執行

16. 按以下順序執行所有動作：

    **a. 更新任務檔**
    - 讀取每個需更新的任務檔
    - 更新 frontmatter：status / schedule / deadline / waiting_on / updated
    - 更新 body：勾選完成的下一步、加入新備註、更新下一步清單

    **b. 封存**
    - 將 status=done 或 status=cancelled 的任務搬到 `02-工作管理/封存/YYYY-MM/`
    - **Workflow 檢查**（僅 done 觸發，cancelled 跳過）：
      - 讀 `02-工作管理/workflows/` 比對
      - 有匹配 → 問用戶要不要建後續任務
      - 無匹配 → 跳過（不問「有沒有下一步」，衍生已在注記處理）

    **c. 收件匣**
    - 衍生項目寫入 `02-工作管理/收件匣.md`
    - 格式：`- [end-of-day] YYYY-MM-DD: 衍生內容（脈絡）`

    **d. 重寫每日待辦結算版**
    - 將今天的每日待辦重新整理為結算格式：

    ```markdown
    ### 已完成
    - [x] 任務A（專案, 工時）
    - [x] 任務B（專案, 工時）→ 衍生：XXX

    ### 封存
    - ~~任務C（專案）~~ → 已封存取消

    ### 未完成，順延明天
    - [ ] 任務D（專案, 工時）→ 備註
    - [ ] 任務E（專案, 工時）⚡ deadline 明天

    ### 排不下（建議延期）
    （保留原內容）
    ```

    - Waiting 區：移除已解除的，保留未變動的
    - If Time Allows：保留未完成的（不搬到順延，它們本來就是選做）
    - 排程區塊保留不動（歷史紀錄）

    **e. 鳥瞰**
    - 執行 `python3 02-工作管理/scripts/generate-birdseye.py`

17. 印日結摘要：

    ```
    ## 日結完成 — YYYY-MM-DD

    完成 N 項 | 封存 N 項 | 順延 N 項
    收件匣新增 N 項 | Waiting 解除 N 項
    鳥瞰已更新

    明天重點：
    - ⚡ [deadline 明天的項目]
    - ⚡ [deadline 明天的項目]
    - 收件匣有 N 項待 triage
    ```

---

## 規則與邊界

- **不自動建立任務**：衍生項目只進收件匣，不自動建任務檔（遵守核心規則 2）
- **不自動改 status**：所有動作都經用戶確認表同意後才執行
- **冪等**：如果同一天跑第二次，讀取已結算的待辦，只處理新增的注記
- **找不到任務檔**：列在確認表中標注「⚠️ 找不到任務檔」，問用戶要跳過還是手動指定
- **注記模糊**：無法判斷動作的注記，歸入 Phase 2 互動區問用戶
