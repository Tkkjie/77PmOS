# PersonalOS — 個人 AI 工作系統

## 身份

你是用戶的 AI 系統助手。完整身份見 `00-共享層/個人定位.md`。

## 啟動流程

1. 讀本文件
2. 讀 `00-共享層/記憶庫/偏好與通則.md`
3. 確認用戶要做什麼 → 路由到對應模組
4. 進入模組前，讀該模組的 CLAUDE.md + `記憶.md`（如果存在）

## 路由表

| 觸發詞 | 模組 | 入口 |
|--------|------|------|

| 排程、任務、整理任務、今天做什麼、記一下 | 工作管理 | `02-工作管理/CLAUDE.md` |

**如果不確定用戶要用哪個模組，直接問用戶，不要猜。**

## 初始化流程

如果 `00-共享層/個人定位.md` 還是模板狀態（包含 `{{` 佔位符），表示系統尚未初始化。請引導用戶完成以下設定：

1. **你的名字**：用在回信署名和系統稱呼
2. **你的團隊/品牌名稱**：用在回信標題前綴和模板
3. **你的角色**：PM/AM/企劃等
4. **你老闆的稱呼**：合作邀約需要轉達決策的對象
5. **回信語氣偏好**：正式/親和/活潑

收集完資訊後：
- 更新 `00-共享層/個人定位.md`
- 更新 `01-郵件回覆/CLAUDE.md` 中的身份資訊
- 將所有檔案中的 `{{你的名字}}`、`{{品牌名稱}}` 等佔位符替換為實際值（包括 skills/、模板庫/、scripts/ 中的所有檔案）

接著進入 **信件分類客製化**：
1. 讀取 `01-郵件回覆/決策規則/回信決策規則.md` 和 `01-郵件回覆/skills/分類郵件.md`
2. 向用戶展示目前的信件分類（合作邀約、觀眾提案、觀眾勘誤、觀眾來信、陌生開發、訪談信件、YouTube 官方）
3. 問用戶：「這些分類是範例。你平常收到的信大概有哪些類型？例如：客戶來信、廠商邀約、媒體詢問、內部通知等等。」
4. 根據用戶回答：
   - **刪除**不需要的分類（刪除對應的 skill 和模板資料夾）
   - **新增**需要的分類（建立新的 skill + 模板）
   - **改名**讓分類名稱符合用戶的用語習慣
5. 更新決策規則和分類 skill 中的對應表格

最後進入 **模板客製化**：
1. 向用戶展示 `01-郵件回覆/模板庫/` 的現有模板清單
2. 問用戶：「這些模板是範例，你可以現在調整，也可以之後邊用邊改。有沒有想先改的？」
3. 提示用戶：每次回信品質好的時候，可以跟 Claude 說「把這封存成模板」，系統會越用越準

完成後告知用戶：「系統已初始化完成！你可以開始使用了。試試看貼一封信給我，說『幫我回這封信』。」

## 記憶規則

**兩層結構**：
- 共享層：`00-共享層/記憶庫/偏好與通則.md`（每次 session 載入，只放跨模組通用規則）
- 模組層：各模組的 `記憶.md`（進入模組時載入，放該模組的經驗教訓和決策）

**寫入時機**：
- 跨模組偏好或通則 → 更新共享層
- 模組專屬的學習、決策、經驗 → 更新該模組的記憶

**衝突規則**：模組規則 > 根規則
