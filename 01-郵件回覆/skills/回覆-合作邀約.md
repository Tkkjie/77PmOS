# Skill：回覆合作邀約

## 用途

處理演講、訪談、活動出席等合作邀約郵件。
**重要：此類郵件必須先產出 Brief 給{{主理人}}確認，不能直接回覆。**

## 流程

### 階段一：提取資訊 + 產出 Brief

1. 從來信中提取關鍵資訊：
   - 邀約方（組織/個人名稱）
   - 聯絡人
   - 活動名稱/主題
   - 活動形式（演講/座談/受訪/其他）
   - 時間/日期
   - 地點（線上/線下，地點在哪）
   - 對{{主理人}}的需求（講什麼、多長、需要準備什麼）
   - 酬勞/條件（如有提及）
   - 截止回覆日期（如有）
3. 讀取 `模板庫/合作邀約/brief-摘要模板.md`
4. 產出 Brief 摘要給用戶，用戶轉交{{主理人}}確認

### 階段二：根據{{主理人}}決策回信

等用戶回報{{主理人}}的決策後：

- **接受** → 讀取 `模板庫/合作邀約/接受-標準回覆.md`，產生接受回信，確認細節
- **婉拒（時間衝突）** → 讀取 `模板庫/合作邀約/婉拒-時間衝突.md`
- **婉拒（主題不符）** → 讀取 `模板庫/合作邀約/婉拒-主題不符.md`
- **需了解更多** → 讀取 `模板庫/合作邀約/需了解更多.md`，列出需要進一步確認的問題
- **其他決策** → 根據{{主理人}}的具體指示產生回信

## 輸出格式（Brief）

```
合作邀約 Brief

邀約方：{組織名稱}
聯絡人：{姓名/職稱}
活動：{活動名稱}
形式：{演講/座談/受訪/其他}
時間：{日期時間}
地點：{線上/線下 + 地點}
需求：{對{{主理人}}的具體需求}
酬勞：{如有提及/未提及}
回覆期限：{如有/未提及}

初步評估：{根據決策規則的初步建議}

請將此 Brief 轉給{{主理人}}確認，回報他的決策後我會產生正式回信。
```

## 輸出格式（回信草稿）

```
回信草稿

主旨：{回信主旨}

---

{回信正文}

---

請審核以上內容，確認後可直接發送。
```
