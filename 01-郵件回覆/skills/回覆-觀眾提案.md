# Skill：回覆觀眾提案

## 用途

處理觀眾提出的內容建議、主題推薦、議題請求等郵件。

## 流程

1. 從來信中提取：
   - 觀眾名稱
   - 提案內容（建議什麼主題/議題）
   - 提案理由（為什麼建議這個主題）
   - 是否附帶資料或參考連結
4. 判斷回覆方向：
   - 這個主題是否已在規劃中
   - 是否與頻道定位相符
   - 提案品質（是否有具體論述）
5. 根據判斷選擇模板，產生回信草稿

## 回覆方向

- **會考慮** → 使用 `模板庫/觀眾提案/感謝-會考慮.md`
- **已有規劃** → 使用 `模板庫/觀眾提案/感謝-已有規劃.md`
- **不太適合** → 感謝提案，委婉說明

## 輸出格式

```
回信草稿（觀眾提案）

提案摘要：{觀眾提了什麼}
回覆方向：{會考慮/已有規劃/不太適合}

主旨：{回信主旨}

---

{回信正文}

---

請審核以上內容，確認後可直接發送。
```

## 寫信提醒

- 用「我會把內容轉達給團隊的企劃研究看看」這類有行動感的轉達語，比「之後團隊會找時間討論」更有誠意
