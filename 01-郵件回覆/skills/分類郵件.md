# Skill：分類郵件

## 用途

判斷來信的郵件類型和子類型，並路由到對應的回覆 Skill。
這是處理任何郵件的第一步，不可跳過。

## 輸入方式

- **直接貼上**：用戶直接貼上郵件文字內容
- **.eml 文件**：用戶提供 .eml 文件路徑，需解析出寄件人、主旨、日期、正文
- **gws CLI**：透過 gws CLI 讀取的郵件資料（含 id、threadId、寄件人、主旨、正文），從流程零自動帶入

## 判斷流程

1. **解析郵件**：提取寄件人、主旨、正文
2. **識別寄件人身份**：
   - YouTube 官方網域 → YouTube 官方
   - 出版社/媒體機構 → 合作邀約或訪談
   - 品牌/公司 → 合作邀約或陌生開發
   - 學生/學校 → 學生訪談邀約
   - 一般個人 → 觀眾提案/勘誤/來信
3. **分析內容意圖** → 確定子類型
4. **輸出結果**

## 輸出格式

```
郵件分類結果

寄件人：{寄件人名稱/郵箱}
主旨：{郵件主旨}
大類：{分類結果}
子類型：{子類型}
判斷理由：{為什麼歸為這個類型}

確認這個分類正確嗎？確認後我會使用對應的回覆 skill 來處理。
```

## 分類後動作

- 用戶確認分類正確 → 讀取對應的回覆 skill 文件並執行
- 用戶指定不同分類 → 按用戶指定的分類執行
- 無法判斷 → 列出最可能的 2-3 個類型，讓用戶選擇
