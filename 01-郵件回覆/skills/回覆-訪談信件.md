# Skill：訪談信件（{{訪談節目名稱}}）

## 用途

處理「{{訪談節目名稱}}」訪談節目的完整信件流程。
這是**主動發出**的信件（{{品牌名稱}}邀請來賓），共分五個階段。

## 五個階段

| 階段 | 名稱 | 說明 | 對應模板 |
|------|------|------|---------|
| 1 | 初次接觸 | 邀請受訪者，介紹節目、說明主題、提供時段 | `模板庫/訪談信件/1-初次接觸.md` |
| 2 | 敲定時程 | 確認錄影時間，告知訪綱寄送時程 | `模板庫/訪談信件/2-敲定時程.md` |
| 3 | 寄出訪綱 | 附訪綱連結、錄影行前資訊（地點/交通/聯繫方式） | `模板庫/訪談信件/3-寄出訪綱.md` |
| 4 | Acopy 確認 | 訪談後寄出 Acopy，請對方確認是否有需剪掉的部分 | `模板庫/訪談信件/4-Acopy確認.md` |
| 5 | 上架通知 | 通知影片上架時間和各平台連結 | `模板庫/訪談信件/5-上架通知.md` |

## 使用流程

1. 用戶告知：受訪者名字、目前信件階段、以及相關必要資訊
2. 讀取對應階段的模板
3. 根據用戶提供的資訊，套入模板產出信件
4. **特殊處理（階段 1）**：需客製「對受訪者的觀察」段落，根據受訪者的背景和近期動態撰寫，並提供過往合作案例連結
5. 輸出可直接複製的信件內容

## 各階段需要的資訊

### 階段 1：初次接觸
- 受訪者姓名/稱呼
- 受訪者的背景（AI 需查找或用戶提供）
- 想聊的主題方向
- 預計討論的內容大綱（3-5 點）
- 可安排的錄影時段（2-3 個選項）
- 希望回信的截止日期
- 企劃署名

### 階段 2：敲定時程
- 受訪者姓名
- 確定的錄影日期時間
- 訪綱預計寄送日期（通常是訪談一週前）
- 擬答回覆截止日期
- 企劃署名

### 階段 3：寄出訪綱
- 受訪者姓名
- 訪綱連結
- 擬答填寫位置（如 D8~D26）
- 擬答截止日期
- 訪談日期、時間
- 地點：{{公司地址}}
- 交通：{{交通說明}}
- 企劃聯繫方式（姓名 + 電話）
- 企劃署名

### 階段 4：Acopy 確認
- 受訪者姓名
- Acopy 檔案連結
- 確認回覆截止日期
- 企劃署名

### 階段 5：上架通知
- 受訪者姓名
- YT 精華版上架日期
- Podcast 完整版上架日期
- 各平台連結（YouTube/Apple Podcast/Spotify/KKBOX）
- 企劃署名

## 寫信規則

1. 使用自然、親切的口吻（用「你們」不要用「您」）
2. 階段 1 的「對受訪者的觀察」要客製化，展現我們對受訪者的了解
3. 階段 1 要根據受訪者性質，提供適合的過往合作案例參考連結
4. 信件最後提醒企劃：**寄出前記得 CC 總編、協調人、PM**
5. 提到曝光時間時要**加預期管理**：說明「{{訪談節目名稱}}」檔期較緊不一定能完全配合，但會盡量協助安排

## 輸出格式

直接輸出可複製的信件內容，最後附上提醒：

```
寄出前提醒：請 CC 總編、協調人、PM
```
