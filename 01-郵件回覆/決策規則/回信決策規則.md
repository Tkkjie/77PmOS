# 回信決策規則

> 通用規則（身份、語氣、開頭格式、回覆時效等）請見 `CLAUDE.md`，此處只放各類型的決策邏輯。

## 合作邀約規則

### 必須 brief {{主理人}}

所有合作邀約必須先產出 Brief，不能擅自接受或拒絕。

### brief必須提取的資訊

- 邀約方（組織名稱）
- 聯絡人
- 活動名稱/主題
- 活動形式（演講/座談/受訪/送書/說書/推薦/授權）
- 時間日期
- 地點（線上/線下）
- 對{{主理人}}的具體需求
- 酬勞/條件
- 回覆期限

### 子類型對應模板

| 子類型 | 判斷依據 | 常用模板 |
|--------|---------|---------|
| 演講/活動邀約 | 邀請演講、座談、出席 | 婉拒-時間衝突 / 婉拒-主題不符 / 接受-標準 / 需了解更多 |
| 出版社送書 | 出版社想寄公關書 | 出版社-接受送書 / 出版社-拒絕送書 |
| 出版社說書 | 出版社邀請製作說書影片 | 出版社-說書合作 |
| 掛名推薦 | 邀請推薦書籍 | 婉拒-推薦掛名（原則上不開放） |
| 影片授權 | 其他平台要授權內容 | 婉拒-影片授權（目前不開放） |
| 學生訪談 | 學生論文/報告訪談 | 婉拒-學生訪談 |
| 商業合作 | 有預算的合作提案 | 接受-標準 / 合作報價單 |

### 預設立場

- **掛名推薦**：原則上不開放 → 直接婉拒
- **影片授權**：已有有償授權合作 → 直接婉拒
- **學生訪談**：通常婉拒 + 推薦{{主理人}}的書
- **學生社團授課**：酬勞低於 **2 萬元** → 直接婉拒（{{主理人}}時間成本考量）；2 萬以上再 brief {{主理人}}
- **其他邀約**：先 brief {{主理人}}

## 觀眾提案規則

### 回覆基調

- 一律感謝觀眾的用心提案
- 即使不會採用，也要讓觀眾感受到被重視

### 子類型對應模板

| 子類型 | 判斷依據 | 對應模板 |
|--------|---------|---------|
| 確實可做 | 主題值得做或正在做 | 感謝-會考慮 |
| 想保持關係 | 現在不能做但未來有機會 | 感謝-已有規劃 |
| 不適合做 | 主題不適合或無法做 | 不適合-婉拒 |
| 已做過 | 已有相關影片 | 已做過（附連結） |
| 想加入團隊 | 觀眾想加入團隊 | 想加入團隊（導向特約作者招募） |

## 觀眾勘誤規則

### 回覆基調

- 感謝觀眾的用心和嚴謹
- 對待事實錯誤要認真處理
- 區分「事實錯誤」「觀點差異」「純補充」

### 子類型對應模板

| 子類型 | 判斷依據 | 對應模板 |
|--------|---------|---------|
| 確實有錯 | 觀眾指出的錯誤成立 | 感謝-確認修正 |
| 純補充資訊 | 補充資料，沒說有錯 | 純補充資訊 |
| 說有錯但不具體 | 說有錯但說不出哪裡，想約面談 | 感謝-說明原因（請對方具體指出） |

## 陌生開發規則

### 判斷標準

- **群發郵件特徵**：模板感強、沒提到{{品牌名稱}} → 建議不回覆
- **有針對性**：明確提到頻道、有具體方案 → 值得評估

### 子類型對應模板

| 子類型 | 判斷依據 | 對應模板 |
|--------|---------|---------|
| 沒預算沒興趣 | 想合作但沒錢也不相關 | 婉拒-標準 |
| 想要曝光 | 沒預算但想免費曝光 | 婉拒-標準 |
| 訴苦/超出能力 | 訴苦或要求超出能力 | 訴苦-建議（同理+建議找誰） |
| 可能有價值 | 有針對性且可能有用 | 有興趣-了解更多 |

## 訪談信件規則（{{訪談節目名稱}}）

訪談信件是**主動發出**的信件（邀請來賓），共分五個階段。
用戶會告知「目前在哪個階段」以及必要資訊。

### 五階段流程

| 階段 | 名稱 | 對應模板 |
|------|------|---------|
| 1 | 初次接觸 | `模板庫/訪談信件/1-初次接觸.md` |
| 2 | 敲定時程 | `模板庫/訪談信件/2-敲定時程.md` |
| 3 | 寄出訪綱 + 行前通知 | `模板庫/訪談信件/3-寄出訪綱.md` |
| 4 | Acopy 確認 | `模板庫/訪談信件/4-Acopy確認.md` |
| 5 | 上架通知 | `模板庫/訪談信件/5-上架通知.md` |

### 寫信規則

- 使用自然、親切的口吻（用「你們」不要用「您」）
- 階段 1 需客製化「對受訪者的觀察」段落
- 階段 1 需提供與受訪者相關的過往合作案例
- **每封信寄出前提醒企劃 CC 總編、協調人、PM**

### 固定資訊

- 訪談地點：{{公司地址}}
- 交通：{{最近交通}}三號出口
- 訪談時長：約一個半小時
- 訪綱寄送時間：通常訪談一週前
- 成品：YT 精華版（約 25 分鐘）+ Podcast 完整版

## YouTube 官方規則

- 政策更新 → 高優先（可能影響頻道運營）
- 合作項目邀請 → 中優先（評估後回覆）
- 一般通知 → 低優先（不一定需要回覆）
- 注意辨別真假（確認寄件人網域）
- 通常用英文回覆
